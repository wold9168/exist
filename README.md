# HITLUG 社团存续讨论

TL;DR: 本仓库为社团存续讨论的议案池。主题为对本社团存续问题的讨论。

## 背景介绍

高校开源技术社团的存续困境，并非偶发的管理失误，而是源于深层的结构性矛盾。究其根本，该现象源于两种深层的结构性矛盾，导致社团普遍面临“其兴也勃焉，其亡也忽焉”的发展态势。

第一，组织使命的“终点效应”导致存在的指向性减弱。此类社团的成立初衷往往源于特定的技术基础设施建设需求（如镜像站搭建、软件源维护等）。这类议题具有明确的目标导向和终点意识——当基础设施建设任务完成，技术难题攻克，社团所依托的必要性便随之消解。在完成既定历史使命后，因缺乏新的议题作为凝聚核心，社团活动便自然进入低谷期。这种“因事设团”的模式，在任务完成之时，便宣告了社团功能性的终结。那么这类社团的低迷和衰微便自然因此发生了。

第二，高边际成本与传承机制的断裂。开源社团的运作是一个高投入的过程，需要核心成员在技术深耕、社群运营及组织管理三个维度持续输出高强度的“情感劳动”与“智力资本”，以换取活跃的技术交流氛围与协作产出。然而，这种输入与输出高度依赖于核心成员的人力资源存量。由于高校学制限制，掌握关键经验与技术的核心成员毕业离校后，往往造成组织记忆与技术能力的断层。即便存在毕业生的远程指导，物理空间的阻隔也难以弥补在场互动缺失带来的影响力衰减。

在此背景下，高校开源社团面临一个深刻的结构性悖论：组织的存续高度依赖具备高专业素养与充沛精力的核心成员。然而，这类成员的高效能往往加速了社团技术使命的完成，从而提前消解了社团存在的现实基础。更为关键的是，此类关键行动者的出现具有高度偶然性，社团组织本身缺乏系统性培养此类人才的能力，也无法通过制度安排确保其精力的持续投入。因此，社团的维系在很大程度上表现为一种对外部人才供给的被动依赖，而非组织内生动力的结果。

套用到本社团而言，本社团完成了我校开源镜像站的建设，完成了组织使命，同时在社团开创前后踊跃的核心成员陆续毕业离校。以上两种结构性矛盾在本社团都有着不同程度的体现，至今而言，已经使得社团又一次进入活动的低迷期。

有鉴于此，我们采纳了社群成员的建议，创设本仓库作为社团治理协作的基础设施。

我们期望以该仓库为载体，实现以下目标：首先，重塑社团活跃度，将线下的松散交流转化为线上可追溯、可沉淀的技术与事务讨论；其次，推动社团的正规化与结构化，通过文档化与版本管理，将隐性的组织经验显性化，以对抗核心成员毕业带来的“组织记忆”流失；再次，实现社团事务的流程化，让活动策划、财务公开、招新考核等事务有章可循，降低后续管理者的边际成本；最后，构建一个公开透明的议案空间，任何成员均可在此提出关于社团发展方向的议案（Proposal），经社区讨论与表决后形成决议。

这一尝试，本质上是对前述结构性矛盾的一次回应。我们希望通过将社团运作“代码化”与“议题化”，寻找打破“其亡也忽焉”的周期律的可能，让社团的生命力不再仅仅维系于几届核心成员的个人热情，而是扎根于开放、协作、可传承的开源文化之中。

期待你的参与，共同定义本社团的下一个篇章。

## 议案池运作机制

一份议案的提出应该经历如下几个流程：

- 在 Issue 中提出 Proposal，经过充分讨论后形成议案文件
- 议案提交到本仓库 PR，在 PR 中被表决
- 被 Merge 的议案视为被接受，否则视为被拒绝

议案的表决至少需要三人参与，表决时间为三天。被接受的 PR 需要在表决期结束时满足 `赞同人数 - 不赞同人数 > 3` 的条件。

出于社团治理考量，无论是否在议案的表决期内，社团主席都可以对议案行使一票否决权。

议案用途的 PR 应当在 PR 标题内标注 `[Proposal]`。
